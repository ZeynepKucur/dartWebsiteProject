Dart Null-aware Operatörler

Null aware operatörler, değerin null olup olmamasına göre işlem yapan operatörlerdir.

  • If-null operatörü (??)
  • Null-aware atama operatörü (??=)
  • Null-aware access operatörü (?.)
  • Null-aware metod çağırma operatörü (?.)
  • Null assertion operatörü (!)
  • Null-aware cascade operaötrü (?..)
  • Null-aware index operatörü (?[])

Dart If-null operatörü (??)

Dart programlama dilinin null değer taşıyor olma olasılığı olan değerler üzerinde işlem yapan operatörlerinden biridir. ?? operatörü solundaki değer null değilse o değeri kullanır, null ise sağındaki değeri kullanır.

Örnek

String? isim;
final ad = isim ?? 'Anonim';
print (ad); //Çıktı: Anonim

Yukarıdaki örnekte isim değişkeni null olduğu için ?? operatörü isim değişkenine sağındaki değer olan 'Anonim'i atar.  ?? operatörü bu şekilde ad değişkeninin hiçbir zaman null olmamasını sağlar.
?? operatörü kullanılan yukarıdaki örnek aşağıdaki koda eşdeğer:

String ad;
if (isim == null){
  ad = 'Anonim';
} else {
  ad = isim;
}

Dart Null-aware assignment operatörü (??=)

Dart’ın sunduğu başka bir null-aware operatör olan ??= ile bir değişkenin null olup olmadığı kontrol edilip null ise ona değer atanabilir.

Örnek

int? sayi;
sayi ?? = 100;
print (sayi); //Çıktı: 100
sayi ??= 10; //derleyici uyarı verir
print(sayi);

sayi değişkeni int? türünde yani null ya da inttüründe tanımlanmıştır ve hiçbir değer atanmamıştır. ??= operatörü sayi değişkeninin null olup olmadığını kontrol eder, null olduğu için 100 değerini atar. print ile yazdırıldığında değerinin 100 olduğu görülür. sayi değişkeni artık null olmadığı için ‘sayi ??= 10;‘ satırında derleyeci uyarı verir ve null olmadığı için 10 değeri atanmaz. print ile yazdırıldığında sayi değişkeninin değerinin değişmediği görülür.

Dart Null-aware access (erişim) operatörü (?.)

Null olabilecek bir nesnenin property ya da metoduna erişimi korumak için kullanılır. Null-aware access ?. operatörü sol tarafındaki null ise null olarak döner değilse erişimi sağlar.

int? yas;
print(yas?.isNegative); //Çıktı: null

Dart dilinin sağladığı null aware access operatörü olmasaydı yukarıdaki kod hata verirdi. ?. operatörü yas değişkeni null olduğu için null değerini döndürür.

Dart Null assertion operatörü ( ! )

Null olabilen bir değişkenin değerinin null olmadığından emin olunuyorsa null assertion operatörü ( ! ) ile Dart’ın o değişkeni null olamayan bir tip olarak görmesi sağlanabilir.

Fonksiyon örneği

bool? yetiskinMi(int? yas) {
  if ( yas == 18) {
    return true;
} else if (yas == 10) {
    return false;
  }
 return false;
}
void main(){
  bool yetiskin = yetiskinMi(10);
}

yetiskinMi fonksiyonu null ya da bool olabilir. yetiskin değişkeni ise null olamayan bool türündedir. Derleyeci bool? türünde bir değişkenin bool türüne atanamayacağı hatasını verir. İşte burada yetiskinMi(10)’un null değer döndürmeyeceğinden emin olunduğundan bu hata null assertion ( ! ) ile düzeltilebilir.

void main(){
  bool yetiskin = yetiskinMi(10)!;
}

Yukarıdaki gibi ! eklendiğinde hata ortadan kalkar. Null assertion operatörünün çok dikkatli kullanılması gerekir çünkü bu operatör ile Dart’ın Null Safety özelliği devre dışı bırakılmış olur. Null olabilen bir expression üzerinde yanlış bir şekilde null assertion operatörü ( ! ) kullanıldığında yapılan hata run-time’da ortaya çıkar.

Dart Null-aware cascade operatörü (?..)

Cascade operatörü aynı nesne üzerinde birden çok işlem yapılmasına olanak sağlar. Null olabilen nesnelerde null-aware cascade operatörü (?..) kullanılabilir.Cascade operatörü aynı nesne üzerinde birden çok işlem yapılmasına olanak sağlar. Null olabilen nesnelerde null-aware cascade operatörü (?..) kullanılabilir.

class Kullanici {
  String? ad;
  int? ad;
}
void main(){
  Kullanici? kullanici1;
  kullanici1
  ?..ad = 'Zeynep'
  ..id = 1;
}
